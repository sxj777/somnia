# Somnia Litepaper 0.1

版本：0.1  
日期：2026-07-05  
官网：https://www.joinsomnia.app

## 1. 项目一句话

Somnia 是一个面向真实目标的 Web3 Dream 发布与社区信号平台。用户可以发布一个 Dream，选择展示位，设定公开展示期限，并通过社区信号、积分和后续审核机制，让值得被看见的目标获得更多关注。

Somnia 不是投资平台，不承诺收益，不出售股权、分红权、债权、代币权益或任何形式的保证回报。

## 2. 我们要解决的问题

很多早期想法、个人目标和小型项目并不缺“幻想”，缺的是一个公开、可追踪、可被社区发现的起点。

传统社交平台的问题是：

- 内容流太快，目标很容易被淹没。
- 点赞只能表达情绪，不能形成持续的建设信号。
- 早期项目很难在没有完整产品前获得第一批关注。
- 用户对“融资”“收益”“投资”的理解容易混在一起，导致风险和误解。

Somnia 想做的是一个更克制的系统：先让 Dream 被发布、展示、记录、排序，再逐步加入积分、审核、里程碑和奖励机制。

## 3. 产品核心机制

### 3.1 发布 Dream

用户连接钱包后，可以发布一个 Dream。每个 Dream 至少包含：

- 标题
- 分类
- 一句话简介
- 建设说明
- 创建者地址
- 展示期限
- 是否选择首页 Spotlight

当前 Alpha 规则：

- 普通 Dream Plaza 展示：10 USDC
- 首页 Spotlight 展示：100 USDC
- 展示周期：3 天

发布费购买的是发布、展示、存储和协议服务，不代表投资、募资或收益权。

### 3.2 Dream Plaza

Dream Plaza 是公开展示区。用户可以浏览正在展示的 Dream，并通过社区 signal 表达支持。

Signal 是社区关注信号，不是投票收益，不产生股权、分红、债权、代币权利或融资承诺。

### 3.3 首页 Spotlight

Spotlight 是更强展示位。选择 Spotlight 的 Dream 可以在首页获得更高曝光。

Spotlight 费用代表展示服务费用，不代表项目质量背书，也不代表 Somnia 对该 Dream 的投资建议。

## 4. 费用分配

Somnia Alpha 采用透明的 50 / 30 / 20 分配模型：

- 50%：Somnia Treasury，用于产品开发、运营、基础设施和社区建设。
- 30%：Dream Vault，用于未来的 Dream 支持、里程碑实验和生态激励。
- 20%：Reviewer Rewards，用于未来审核者、评审者和贡献者奖励。

Alpha 阶段的 Dream Vault 和 Reviewer Rewards 是产品机制预留，不代表任何用户可获得固定收益。

## 5. Somnia Points 积分系统

积分系统是 Somnia 融资前必须上线的核心模块。原因是：在对外融资或扩大社区前，Somnia 需要先有一个可体验、可验证、可增长的用户行为系统。

### 5.1 积分是什么

Somnia Points 是平台内的声誉和参与度积分，用来记录用户对 Dream 生态的参与。

积分可以用于：

- 展示用户活跃度
- 排行榜和等级
- Dream Plaza 排序权重
- 审核者资格门槛
- Beta 功能白名单
- 未来社区治理实验的参考信号

### 5.2 积分不是什么

Somnia Points 不是代币，不可转让，不可交易，不承诺兑换，不代表股权、分红、债权、收益权、空投资格或未来代币权利。

Somnia 可以根据反作弊、产品调整、法律合规和社区治理需要，修改积分规则、冻结异常积分或重置测试期积分。

### 5.3 积分 MVP 规则

第一版积分系统应尽量简单，先让用户能看到积分增长，再逐步升级。

建议 MVP 规则：

| 行为 | 积分 |
| --- | ---: |
| 首次连接钱包并创建基础身份 | +20 |
| 发布一个有效 Dream | +100 |
| 发布后补充一次进展更新 | +30 |
| 给其他 Dream 发送 signal | +5 |
| 自己的 Dream 收到一个有效 signal | +2 |
| Dream 被标记为精选候选 | +50 |

建议限制：

- 给他人 signal 每日最多计 50 分。
- 单个 Dream 收到 signal 带来的积分最多计 200 分。
- 自己不能通过重复钱包刷分。
- 被判定为垃圾内容、诈骗内容或恶意刷分的 Dream，可以扣分或清零。

### 5.4 积分系统第一阶段页面

融资前建议至少上线以下页面或模块：

- 个人积分卡：显示钱包地址、总积分、等级、最近积分记录。
- Dream 卡片积分信息：显示该 Dream 收到的 signal 和贡献积分。
- 排行榜：展示活跃创建者、热门 Dream、社区贡献者。
- 积分规则说明：明确积分不是投资权益。

## 6. 用户角色

### Dream Creator

发布 Dream 的用户。Creator 通过创建、更新和推动 Dream 获得积分和曝光。

### Supporter

浏览 Dream 并发送 signal 的用户。Supporter 通过发现和支持有潜力的 Dream 获得参与积分。

### Reviewer

未来参与审核、标记、分类和评估 Dream 的贡献者。Reviewer 不在 Alpha 第一阶段开放，需要积分、历史行为和人工筛选作为门槛。

### Treasury Operator

负责协议运营、基础设施、内容维护和资金分配透明度的角色。正式阶段建议使用多签钱包管理。

## 7. 路线图

### Phase 0：正式官网

- 官网上线
- Logo / favicon 上线
- 基础 Dream 发布叙事完成
- 风险提示和非投资声明上线

### Phase 1：Litepaper 与积分 MVP

- 发布 Litepaper 0.1
- 上线 Somnia Points
- 上线用户积分卡
- 上线排行榜
- 将 signal 与积分联动

### Phase 2：Dream 审核与里程碑

- 引入 Reviewer 角色
- 增加 Dream 进展更新
- 增加里程碑状态
- 将 Dream Vault 用于更明确的支持实验

### Phase 3：主网准备

- 完成合约审计
- 完成法律审查
- 多签管理 Treasury、Dream Vault、Reviewer Rewards
- 完成正式服务条款和隐私政策

## 8. 风险提示

Somnia 仍处于 Alpha 阶段。用户应理解钱包交易、测试网行为、链上操作不可逆、内容公开展示和智能合约风险。

用户不应发布隐私信息、违法内容、误导性融资信息或承诺收益的内容。

Somnia 网站、Litepaper、积分规则和产品界面中的任何内容都不构成金融建议、投资建议、法律建议或融资承诺。任何主网、融资、代币或治理相关动作，都应在完成法律审查和风险披露后进行。

## 9. 当前最重要的判断

Somnia 融资前不应该只讲愿景。更稳的路径是：

1. 官网已经上线。
2. Litepaper 先定义边界。
3. 积分系统尽快上线，让用户行为开始沉淀。
4. 再围绕真实数据、真实用户和真实参与度展开融资叙事。

这会比直接讲“融资模式”更可信，也更容易降低合规和认知风险。
